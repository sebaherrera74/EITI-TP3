4 ## Overview
5 
6 The module_generator plugin adds a pair of new commands to Ceedling, allowing
7 you to make or remove modules according to predefined templates. WIth a single call,
8 Ceedling can generate a source, header, and test file for a new module. If given a
9 pattern, it can even create a series of submodules to support specific design patterns.
10 Finally, it can just as easily remove related modules, avoiding the need to delete
11 each individually.
12 
13 Let's say, for example, that you want to create a single module named `MadScience`.
14 
15 ```
16 ceedling module:create[MadScience]
17 ```
18 
19 It says we're speaking to the module plugin, and we want to create a new module. The
20 name of that module is between the brackets. It will keep this case, unless you have
21 specified a different default (see configuration). It will create three files:
22 `MadScience.c`, `MadScience.h`, and `TestMadScience.c`. *NOTE* that it is important that
23 there are no spaces between the brackets. We know, it's annoying... but it's the rules.
24 
25 You can also create an entire pattern of files. To do that, just add a second argument
26 to the pattern ID. Something like this:
27 
28 ```
29 ceedling module:create[SecretLair,mch]
30 ```
31 
32 In this example, we'd create 9 files total: 3 headers, 3 source files, and 3 test files. These
33 files would be named `SecretLairModel`, `SecretLairConductor`, and `SecretLairHardware`. Isn't
34 that nice?
35 
36 Similarly, you can create stubs for all functions in a header file just by making a single call
37 to your handy `stub` feature, like this:
38 
39 ```
40 ceedling module:stub[SecretLair]
41 ```
42 
43 This call will look in SecretLair.h and will generate a file SecretLair.c that contains a stub
44 for each function declared in the header! Even better, if SecretLair.c already exists, it will
45 add only new functions, leaving your existing calls alone so that it doesn't cause any problems.

47 ## Configuration
48 
49 Enable the plugin in your project.yml by adding `module_generator`
50 to the list of enabled plugins.
51 
52 Then, like much of Ceedling, you can just run as-is with the defaults, or you can override those
53 defaults for your own needs. For example, new source and header files will be automatically
54 placed in the `src/` folder while tests will go in the `test/` folder. That's great if your project
55 follows the default ceedling structure... but what if you have a different structure?
56 
57 ```
58 :module_generator:
59  :project_root: ./
60  :source_root: source/
61  :inc_root: includes/
62  :test_root: tests/
63 ```
64 
65 Now I've redirected the location where modules are going to be generated.
66 
67 ### Includes
68 
69 You can make it so that all of your files are generated with a standard include list. This is done
70 by adding to the `:includes` array. For example:
71 
72 ```
73 :module_generator:
74  :includes:
75  :tst:
76  - defs.h
77  - board.h
78  :src:
79  - board.h
80 ```
81 
82 ### Boilerplates
83 
84 You can specify the actual boilerplate used for each of your files. This is the handy place to
85 put that corporate copyright notice (or maybe a copyleft notice, if that's your perference?)
86 
87 ```
88 :module_generator:
89  :boilerplates: |
90  /***************************
91  * This file is Awesome. *
92  * That is All. *
93  ***************************/
94 ```
95 
96 ### Test Defines
97 
98 You can specify the "#ifdef TEST" at the top of the test files with a custom define.
99 This example will put a "#ifdef CEEDLING_TEST" at the top of the test files.
100 
101 ```
102 :module_generator:
103  :test_define: CEEDLING_TEST
104 ```
105 
106 ### Naming Convention
107 
108 Finally, you can force a particular naming convention. Even if someone calls the generator
109 with something like `MyNewModule`, if they have the naming convention set to `:caps`, it will
110 generate files like `MY_NEW_MODULE.c`. This keeps everyone on your team behaving the same way.
111 
112 Your options are as follows:
113 
114  - `:bumpy` - BumpyFilesLooksLikeSo
115  - `:camel` - camelFilesAreSimilarButStartLow
116  - `:snake` - snake_case_is_all_lower_and_uses_underscores
117  - `:caps` - CAPS_FEELS_LIKE_YOU_ARE_SCREAMING
